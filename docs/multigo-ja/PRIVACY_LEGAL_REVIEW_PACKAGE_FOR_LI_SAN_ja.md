### 2.1 重要な質問

**質問1: スコープの適切性**
```
現在の要件は以下をカバー:
- GDPR（EU一般データ保護規則）
- PIPEDA（カナダ）
- HIPAA（米国医療）
- CCPA（カリフォルニア）

質問: [場所]のパイロットプロジェクトに実際に適用される規制は？
推奨: 適用される規制のみに簡略化？
```

**質問2: HIPAAの適用性**
```
現在の文書には広範なHIPAA要件（セクション15）が含まれています。

質問: パイロットプロジェクトはHIPAAの「対象事業体」に該当するか？
コンテキスト: ロボットは入居者を搬送（医療状態がある可能性）するが、電子健康記録にアクセス/送信しない。

```

**質問3: 同意メカニズム**
```
要件ではサービス前に入居者から明示的な同意を取得することを指定。

質問: 施設が入居者に代わって同意を提供できるか（例: サービス契約で）？
```

## 3. 現在の要件の概要

### 3.1 データ保護原則（7要件）

**REQ-PRIV-001からREQ-PRIV-007:**
- 適法性、公正性、透明性
- 目的制限
- データ最小化
- 正確性
- 保存期間制限
- 完全性と機密性
- 説明責任

**法的レビュー必要:** すべての7原則がパイロットに法的に必要か、それともベストプラクティスか？

### 3.2 収集されるPIIカテゴリー

**入居者データ:**
- 名前、年齢、連絡先情報
- 医療状態（アレルギー、移動制限）
- 位置履歴（搬送場所）
- サービス使用ログ

**オペレーターデータ:**
- 名前、メール、役割
- ログインタイムスタンプ
- 実行されたアクション（監査証跡）

**カメラデータ:**
- リアルタイム画像（ナビゲーション用）
- 一時的に保存（約10分）
- 偶発的に人の顔を捉える可能性

**法的レビュー必要:**
- サービスに「必要」なデータはどれか？
- 明示的な同意が必要なデータはどれか？
- カメラデータは「監視」と見なされるか？

### 3.3 データ主体の権利（GDPR第15-22条）

**現在指定されているもの:**
1. アクセス権（データのコピーを提供）
2. 訂正権（不正確なデータを修正）
3. 削除権（「忘れられる権利」）
4. 処理制限権
5. データポータビリティ権
6. 異議権
7. 自動意思決定に関連する権利

**法的レビュー必要:**
- パイロットプロジェクトに適用される権利はどれか？
- 一部の権利をパイロット後に延期できるか？
- リクエストに対応するタイムラインは？

### 3.4 同意要件

**現在の要件:**
- データ収集前の明示的でインフォームドな同意
- 各目的に対する個別の同意
- 簡単な撤回メカニズム
- 同意の記録を維持

**法的レビュー必要:**
- 施設がすべての入居者に代わって同意契約に署名できるか？
- 入居者ごとの個別同意？
- 入居者に同意能力がない場合（認知症）はどうするか？
- 家族が入居者に代わって同意できるか？

### 3.5 データ保持期間

**現在指定されているもの:**
| データタイプ | 保持期間 |
|-----------|------------------|
| テレメトリーデータ | 30日 |
| カメラ画像 | 10分（リアルタイムのみ） |
| ミッションログ | 1年 |
| 入居者プロファイル | サービス期間 + 6ヶ月 |
| 監査ログ | 1年 |

**法的レビュー必要:**
- これらは法的に必要な最小値か？
- デバッグ/改善のためにより長く保持できるか？
- 何が削除をトリガーするか（サービス終了、入居者の要求）？

## 4. パイロットプロジェクトの考慮事項

### 4.1 現在の状況

**規模:**
- 1-2台の車両
- 単一施設（約50-100名の入居者）
- 限定された地理（施設敷地内のみ）
- 管理された環境

**簡略化の機会:**
- 国際コンプライアンスを削除（単一国の場合）
- HIPAA削除（適用されない場合）
- 子供のプライバシー削除（高齢者ユーザーのみ）
- データ主体の権利を簡略化（コア権利のみ実装）

### 4.2 提案される簡略化（法的承認待ち）

**現状:** 1,124行、包括的なエンタープライズコンプライアンス
**目標:** 約300行、パイロットに適したコンプライアンス

**保持:**
- データ最小化（必要なデータのみ収集）
- 同意メカニズム（法務が決定した方法）
- 基本セキュリティ（暗号化、アクセス制御）
- データ保持/削除ポリシー
- 侵害通知（法律で必要な場合）

**削除/延期:**
- HIPAA要件（適用されない場合）
- 国境を越えた転送ルール（データがローカルにとどまる場合）
- 子供のプライバシー（関連しない場合）
- 高度なデータ主体の権利（パイロット後に延期）
- 国際規制（適用される管轄区域のみ保持）

### 4.3 リスク評価

**低リスク（パイロット範囲）:**
- 小規模ユーザーベース（<100人）
- 管理された環境（単一施設）
- 金融取引なし
- 医療診断/治療なし
- 限定された地理的エリア

**中リスク:**
- カメラが顔を捉える可能性（監視懸念）
- 位置追跡（プライバシー懸念）
- 高齢者人口（脆弱グループ）

**高リスク（完全なコンプライアンスが必要）:**
- 複数施設展開
- 国境を越えたデータ転送
- 電子健康記録との統合
- 認証用の顔認識

**法的レビュー必要:** リスクレベルと必要なコンプライアンス措置を確認
