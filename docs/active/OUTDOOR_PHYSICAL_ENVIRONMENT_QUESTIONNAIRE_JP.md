## 1. 地形と地表面

### Q1.1: 路面タイプの分布
**質問:** ルートの各路面タイプの割合は?

**測定オプション:**
- [ ] 滑らかなアスファルト/コンクリート: ____%
- [ ] 粗いアスファルト(ひび割れ): ____%
- [ ] 舗装タイル/レンガ: ____%
- [ ] 砂利道: ____%
- [ ] 芝生(手入れされた): ____%
- [ ] 土の道(固められた): ____%
- [ ] 混合(遷移): ____%

**影響:**
- **ハードウェア:**
  - 滑らか(>80%) → メカナムが機能する可能性(アップグレード付き)
  - 粗い/砂利(>20%) → **スワーブドライブまたはディファレンシャル必須**
  - 混合 → **適応サスペンション**または大きなホイール径が必要
- **ソフトウェア:**
  - 滑らか → 標準Nav2コストマップ(5cm解像度でOK)
  - 粗い → **コストマップ解像度を2cmに増加**(穴を検出)
  - 砂利 → **ポイントクラウドフィルタリング**が必要(地面のクラッターを無視)
- **コスト影響:**
  - 滑らか: +$0 (現在のホイールでアップグレード可能)
  - 粗い: +$1500 (屋外グレードメカナム)
  - 混合: +$5000-7000 (スワーブドライブ必須)

**優先度:** 🔴 **クリティカル** - ホイールシステム全体の選択を決定

## 2. 傾斜と勾配

### Q2.1: 通常ルートの最大傾斜(既に質問済み)
**質問:** 通常ルートの最大傾斜角度は?

**オプション:**
- [ ] 0-2° (本質的に平坦)
- [ ] 2-5° (緩やかな傾斜、駐車場で一般的)
- [ ] 5-10° (顕著な坂)
- [ ] 10-15° (急、助けが必要な可能性)
- [ ] >15° (非常に急、車椅子ユーザーが苦労)

**影響:**
- **ハードウェア:**
  - 0-2° → メカナムOK(チューニング付き)
  - 2-5° → **メカナム苦労**(横方向ドリフト)
  - >5° → **スワーブドライブ必須**(より良いトラクション)
  - >10° → **モーターパワーアップグレード**が必要(+20-30%トルク)
- **ソフトウェア:**
  - >2° → ドッキングでの**傾斜補償**(IMUベース)
  - >5° → **傾斜対応経路計画**(バッテリー低下時は登り坂を避ける)
  - PID制御での**重力補償**
- **バッテリー:**
  - +5°ごとに → **約20%多くのバッテリー消費**
  - >10° → **より大きなバッテリー**が必要な可能性 (+$500、+2kg重量)
- **ドッキング:**
  - 0-1° → ±1mm達成可能
  - 2-5° → ±5-10mm現実的
  - >5° → **ドッキング失敗の可能性**(アプローチ中にスライド)

**優先度:** 🔴 **クリティカル** - システム設計への最大影響
